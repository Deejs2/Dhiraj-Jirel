Understanding Dependency Injection in Spring Boot

July 26, 2024 by Dhiraj Jirel

Dependency Injection (DI) is a fundamental concept in Spring Boot that promotes loose coupling between components. By using DI, you can manage the dependencies of your application more effectively, making it easier to test and maintain. In this blog post, we will explore how dependency injection works in Spring Boot and provide practical examples.

What is Dependency Injection?

Dependency Injection is a design pattern used to implement IoC (Inversion of Control). Instead of creating dependencies directly within a class, you inject them from the outside. This approach allows for better modularity and easier testing of your code.

Benefits of Dependency Injection

  • Improves code modularity and reusability.
  • Makes code easier to test.
  • Reduces tight coupling between components.
  • Enhances code maintainability and flexibility.

Setting Up Dependency Injection in Spring Boot

1. Create a Service Interface


package com.example.demo.service;

public interface MessageService {
String getMessage();
}

2. Implement the Service Interface


package com.example.demo.service;

import org.springframework.stereotype.Service;

@Service
public class EmailMessageService implements MessageService {
@Override
public String getMessage() {
    return "Email message service";
}
}

3. Create a Controller to Use the Service


package com.example.demo.controller;

import com.example.demo.service.MessageService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/message")
public class MessageController {

private final MessageService messageService;

@Autowired
public MessageController(MessageService messageService) {
    this.messageService = messageService;
}

@GetMapping
public String getMessage() {
    return messageService.getMessage();
}
}

4. Run Your Application


package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class DemoApplication {
public static void main(String[] args) {
    SpringApplication.run(DemoApplication.class, args);
}
}
